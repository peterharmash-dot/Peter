# About me

So, a bit more about me than the README.

I'm Peter. By title I'm a design engineer, but really I just like sitting in the gap between how something looks and how it's built. A design that looks great but falls apart in code bugs me, and so does clean code wrapped around a confusing interface. I keep ending up somewhere in the middle.

How I usually work: start with a messy idea, sketch or prototype it fast, then spend way longer than I'd admit polishing the details. I'd rather ship one thing that feels right than five that are "fine."

The stack I reach for is boring on purpose — HTML, CSS, JavaScript, and a design system to keep things consistent. My big goal this year is to actually finish a personal project for once.

A couple of things I keep coming back to:

- Simple beats clever, almost every time
- The details are the product, not an afterthought
- If people can't figure out how to use it, it doesn't matter how nice it looks
